What exists so far — Stages 1–3

A passwordless CI path: code on GitHub, GitHub Actions assumes an IAM role by OIDC, and pushes a container image to a private ECR registry. No long-lived AWS keys are stored anywhere.

1 · OIDC handshake

Actions presents a signed token. IAM checks the provider trust and the role's sub condition, then hands back temporary credentials. No stored keys.

2 · Push to ECR

With those temp creds, Actions runs docker build and pushes llm-app:<sha> into the private registry.

Next · Stage 4

Amazon EKS pulls this image and runs it. It is the first resource that bills hourly, so it goes up and comes down in one sitting.
